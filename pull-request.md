## Pull Request (PR): как предложить изменения


PR — это запрос на слияние вашей ветки с основной (`main` или `master`).


### Зачем нужны PR?

- **Рецензирование**: коллеги проверяют код перед вливанием.  
- **Обсуждение**: можно обсудить изменения в комментариях.  
- **Автотесты**: CI/CD проверяет код на ошибки.  
- **История**: все изменения документированы.


### Как создать PR


1. Залейте свою ветку на GitHub:  
   ```bash
   git push origin ваша-ветка

Перейдите на страницу репозитория GitHub.

Нажмите «Compare & pull request» (появится баннер с предложением).

Заполните:

Заголовок: кратко о сути изменений (например, add: форма регистрации).

Описание: что и зачем изменено. Можно добавить скриншоты.

Выберите целевую ветку (main).

Нажмите «Create pull request».
